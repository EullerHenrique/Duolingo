O que é o Angular CLI (ng)

O Angular CLI é uma ferramenta open source desenvolvida pelo próprio time do Angular e é utilizado para facilitar
a criação de componentes, classes, services e outros.

Comando para instalar

    sudo npm install -g @angular/cli@1.1.0

Criar um projeto

    ng new app1

Gerar o build (executar o webpack) e gerar um servidor remoto
    ng server ou ng serve

Criar um componente
  ng generate component <nome>
  ou
  ng g c <nome>
  ou
  ng g c <nome> --spec=false     spec -> Arquivo utilizado para testes
                                 spec=false -> O arquivo spec não será criado

Instalar bootstrap
  npm install bootstrap


O que é webpack?

    Webpack é um module bundler (empacotador de módulos) para JavaScript, em outras palavras, ele junta os arquivos JS
    (e também outros formatos) da sua aplicação (seja arquivos seus ou dependências externas) em um arquivo só
    (ou mais de um), de forma otimizada. Os arquivos são unificados na ordem certa e sem duplicação.

Como o webpack sabe quais arquivos incluir e em qual ordem?
    Ele monta um grafo de dependências.

    Partindo de um arquivo inicial (entrada), ele verifica quais outros arquivos são importados por este, e os
    adiciona ao grafo. Para cada um desses ele também verifica quais eles importam, assim sucessivamente até que
    o grafo contenha todos os arquivos (módulos) necessários para a aplicação.



chunks (pedaços) / bundles (pacotes)

    polyfills ->
        Integra todos os polyfills da aplicação
            Um polyfill é um pedaço de código (geralmente JavaScript na Web) usado para fornecer funcionalidades modernas
            em navegadores mais antigos que não o suportam nativamente
    main ->
        Integra todos os módulos e componentes da aplicação
    styles ->
        Integra todos os arquivos de estilo da aplicação
    vendor ->
        Integra todas as bibliotecas angular da aplicação
    inline ->
        Integra os utilitários do webpack necessários para realizar o bootstraping da aplicação

        Auxilia no carregamento dos demais scripts da aplicação.




Gerar o build (executar o webpack) de desenvolvimento (Destinada ao programador)

   ng build ou ng build --dev

Obs: Como essa build é destinada ao desenvolvimento, ela possui
arquivos .map, tais recursos são utilizados para o auxiliar o processo de debug (identificar e corrigir bugs/erros)

Gerar o build (executar o webpack) de produção (Destinada à publicação)

  ng build --prod --aot=false


  Diferenciais:

  * Os nomes dos arquivos são concatenados com um timeStamp, ou seja, o tempo ( em milissegundos desde 1970 )
  no momento em que o arquivo foi gerado. Isso é feito para que se o projeto for atualizado, o navegador carregue
  os novos arquivos, e não os arquivos armazenados no cache.

  * O bundle destinado a armazenar arquivos css deixa de possuir a extensão .js para ter a extensão .css

  * Todos os códigos são minificados, ou seja, espaços são removidos, quebras de linha e comentários.
  Dessa maneira, o tamanho de cada arquivo é reduzido

  * O código passa por um processo de ofuscagem, ou seja, os nomes das variáveis são reduzidas o máximo possível.
  Dessa maneira, os tamanho de cada arquivo é reduzido. Ex: let tamanho12345 -> let a

  * O debug se torna praticamente impossível



  Obs: O build será armazenado na pasta dist



  Deploy (implantar, colocar em posição, disponibilizar para uso, "colocar no ar")

    Deploy local via servidor HTTP

      * Rode o comando

          ng build --prod --aot=false --base-href <"Pasta-raiz">

      * Como a pasta raiz foi trocada, os caminho para as imagens devem ser atualizados
          Nos caminhos que comemçam com "/assets...", retire a barra

      * Coloque a pasta dist na pasta pública (public_html) do servidor HTTP (ex: Apache)


        Projeto acessível por meio de dois servidores:
            1)Servidor que utiliza o ip do roteador (Qualquer dispositivo pode acessar esse servidor)
            2)Servidor que utiliza o ip do computador (Somente o dispositivo atual pode acessar esse servidor (localhost))



    Deploy local via pacote HTTP-server
        Instalar

          npm install http-server -g

          Obs: -g -> global, o http-server pode ser utilizado em qualquer local do sistema operacional
          por meio da linha de comando

        Gerar http-server

           Rode o comando http-server dentro da pasta dist
              Dois servidores serão gerados
                1)Servidor que utiliza o ip do roteador (Qualquer dispositivo pode acessar esse servidor)
                2)Servidor que utiliza o ip do computador (Somente o dispositivo atual pode acessar esse servidor (localhost))




      Build para o git hub pages

          Use a CLI Angular com o --base-href sinalizador para criar seu projeto e defina o local href base correto:

            ng build --prod --ato=false --base-href "<repo-name>"

            ex:  ng build --prod --ato=false --base-href ngh https://eullerhenrique.github.io/Projeto_Duolingo/


    Deploy para o git hub pages

        Instale o angular-cli-ghpages (ngh) globalmente:

            sudo npm install -g angular-cli-ghpages

         Execute o angular-cli-ghpages
              sudo ngh
